Дизайн-документ игры "My Game"

# Идея

## 1.1 Жанр

2D платформер с элементами головоломки и управления временем.

---

## 1.2 Игровой мир

Игровой мир — это разрушенная станция управления временем.

Особенности мира:

- разделён на “сектора времени” (прошлое, настоящее, будущее)
- одни и те же локации выглядят по-разному в разных временных состояниях
- некоторые объекты существуют только в определённом времени

Изначально мир:

- нестабилен  
- содержит разрывы времени (гличи)  
- части уровней недоступны  

По мере прохождения:

- игрок стабилизирует временные зоны  
- получает доступ к новым состояниям времени  
- изменяет окружение через прошлое  

На поздних этапах:

- временные линии начинают пересекаться  
- появляются парадоксы (объекты дублируются, исчезают)  
- уровни становятся нелогичными и сложными  

---

## 1.3 Сюжет

Игрок — оператор станции, который потерял контроль над системой времени.

Сюжет подаётся через:

- записи предыдущих операторов  
- визуальные изменения уровней  
- “эхо” прошлых действий игрока  

Ход сюжета:

- герой просыпается на разрушенной станции  
- обнаруживает способность переключать время  
- пытается восстановить систему  
- сталкивается с последствиями изменений (парадоксы)  
- понимает, что сам стал причиной катастрофы  

Финал:

- выбор: полностью остановить время или зациклить его  

Основная тема:

- контроль vs последствия  
- ошибки, которые нельзя отменить  

---

## 1.4 Персонаж(-и) и враги

### Главный персонаж

Оператор станции.

Особенности:

- не разговаривает напрямую  
- его прошлые версии появляются как “эхо”  
- может взаимодействовать с самим собой из прошлого  

---

### Враги

Враги — временные аномалии.

Типы:

- **Эхо-двойники** — повторяют действия игрока с задержкой  
- **Разломы** — области, где нарушена физика  
- **Парадоксы** — враги, появляющиеся при ошибках игрока  

Особенность:

- многие враги — результат действий игрока  

---

### Дополнительные элементы

- Всплывающие записи — дают подсказки и раскрывают сюжет  
- Объекты времени — платформы и механизмы, зависящие от временного слоя  
- Временные якоря — точки сохранения и стабилизации времени

# Геймплей

## 2.1 Логика игры

Основная цель игрока — пройти уровень, используя простое переключение времени и взаимодействие с объектами.

Игроку необходимо:

- перемещаться по уровню (движение влево/вправо, прыжки)
- переключаться между двумя состояниями времени (прошлое / настоящее)
- активировать объекты (кнопки, двери)
- избегать препятствий

Ключевая логика:

- действия в прошлом влияют на настоящее  
- объекты могут существовать только в одном времени  
- для прохождения нужно правильно выбрать момент переключения  

Пример:

- в прошлом игрок нажимает кнопку  
- в настоящем открывается дверь  

Игровой процесс строится на:

- простых логических задачах  
- наблюдении за изменениями  
- последовательности действий  

---

## 2.2 Игровые механики

### Основные механики

**Переключение времени**

- игрок может переключаться между прошлым и настоящим
- изменяется:
  - расположение платформ  
  - состояние объектов  
  - доступность путей  

---

**Платформинг**

- передвижение по платформам
- прыжки через препятствия
- избегание ловушек

---

**Взаимодействие с объектами**

Типы объектов:

- кнопки (открывают двери)
- двери (блокируют путь)
- платформы (могут исчезать/появляться в разных временах)

---

### Упрощённые дополнительные механики

**Ограниченное использование времени**

- переключение можно использовать без ограничений
- но на некоторых уровнях требуется точный тайминг

---

**Простые враги (по желанию)**

- двигаются по одному маршруту
- не меняют поведение
- их нужно просто обходить

---

## 2.3 Уровни игры

Структура уровней линейная, с постепенным усложнением:

---

### 1. Обучающие уровни

Цель:

- научить базовым действиям  

Особенности:

- простые платформы  
- одно переключение времени  
- отсутствие врагов  

---

### 2. Простые уровни

Цель:

- закрепить механику времени  

Особенности:

- комбинация кнопок и дверей  
- необходимость переключаться между временем  

---

### 3. Средние уровни

Цель:

- развитие логики игрока  

Особенности:

- несколько объектов на уровне  
- нужно выполнять действия в правильном порядке  

---

### 4. Усложнённые уровни

Цель:

- объединение всех механик  

Особенности:

- более сложные маршруты  
- добавление простых врагов  
- требуется точность в действиях  

---

### Финальные уровни

Особенности:

- проверка всех полученных навыков  
- более длинные уровни  
- несколько последовательных задач  

Финал:

- игрок проходит последний уровень, используя все изученные механики
